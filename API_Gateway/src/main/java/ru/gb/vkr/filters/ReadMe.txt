Логика для каждого фильтра:

1. AuthenticationFilter: Проверяет аутентификацию пользователя и устанавливает аутентифицированного
пользователя в контексте безопасности.
2. AuthorizationFilter: Проверяет авторизацию пользователя и устанавливает соответствующий код ответа,
если пользователь не авторизован.
3. RateLimitingFilter: Использует библиотеку Guava для ограничения скорости запросов и устанавливает
код ответа 429 Too Many Requests, если лимит превышен.
